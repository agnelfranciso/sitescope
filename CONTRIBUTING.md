# Contributing to SiteScope 🚀

Thank you for your interest in contributing to **SiteScope**! This tool is built by developers, for developers, and your help is always welcome.

## 🛠 How Can You Contribute?

### 1. Reporting Bugs
- Use the [GitHub Issue Tracker](https://github.com/your-username/sitescope/issues) to report visual or functional bugs. 
- Please include your Node.js version and the OS you are using.

### 2. Suggesting Features
- We love new ideas! If you want a specific feature (like "A11y Checks" or "SEO Metadata Analysis"), open an issue labeled `enhancement`.

### 3. Sharing Your Ideas
- We love new ideas! If you want a specific feature (like "A11y Checks" or "SEO Metadata Analysis"), open an issue labeled `enhancement`.

## 🔒 Note on Code Contributions
At this stage, **SiteScope is a closed-contribution project**. We are not currently accepting external Pull Requests (PRs) as we want to maintain the core architectural vision. 

However, we **greatly appreciate** your bug reports and feature suggestions in the Issue tracker!

## 🧪 Local Development

Since SiteScope is a terminal-only CLI tool, the easiest way to test your changes is to:
1.  Clone the repo.
2.  Run `npm install`.
3.  Link your local version: `npm link`.
4.  Run `sitescope` from any test directory.

## 📜 Code of Conduct
Please be respectful and helpful to others in the community. Let's build the best "Vibecoding" tool together!
